# Лекция 8. Interprocedural

## Пример программы, для которой контекстно-чувствительный анализ знаков требует коэффициент k > 1 (слайд 37)

### Пример программы и объяснение

Рассмотрим пример:

```tip
f(z) {
    return z;
}

h(a) {
    return f(a);
}

main() {
    var x, y;

    x = h(1);
    y = h(-1);

    return x;
}
```

Здесь функция `f` просто возвращает свой аргумент. Функция `h` вызывает `f` в одном и том же месте программы:

```tip
return f(a);
```

В функции `main` функция `h` вызывается два раза с разными аргументами:

```tip
x = h(1);
y = h(-1);
```

Первый вызов передаёт положительное значение, второй — отрицательное.

Если использовать контекстную чувствительность с $k = 1$, то контекст функции `f` определяется только последним местом вызова. Но оба вызова `f` происходят из одного и того же места внутри функции `h`, поэтому анализ объединяет два случая:

```text
z -> +
z -> -
```

После объединения получается:

```text
z -> ?
```

Следовательно, результат функции `f` тоже становится неизвестным:

```text
result -> ?
```

Из-за этого анализ не может точно понять, что в первом вызове `h(1)` результат положительный, а во втором вызове `h(-1)` результат отрицательный.

Если же использовать $k = 2$, то контекст будет учитывать не только вызов `f` внутри `h`, но и место, из которого была вызвана сама функция `h`.

Тогда анализ сможет различить два контекста:

```text
main -> h(1) -> f(a)
main -> h(-1) -> f(a)
```

В первом контексте:

```text
z -> +
result -> +
```

Во втором контексте:

```text
z -> -
result -> -
```

Таким образом, для этой программы глубины контекста $k = 1$ недостаточно, потому что она склеивает разные вызовы `f` внутри `h`. Глубина $k = 2$ позволяет сохранить информацию о том, откуда была вызвана `h`, и поэтому анализ становится точнее.

### Пример решётки для анализа

Для анализа знаков используется решётка значений переменных:

```text
      ?
     / \
    +   -
     \ /
      0
```

Где:

* `+` — положительное значение,
* `-` — отрицательное значение,
* `0` — ноль,
* `?` — неизвестное значение (результат объединения разных знаков).

Операция объединения (join) работает так:

```text
+ ⊔ - = ?
+ ⊔ + = +
- ⊔ - = -
```

В контекстно-чувствительном анализе используется отображение:

```text
Contexts -> (Variables -> Signs)
```

То есть для каждого контекста вызова хранится информация о знаках переменных.

При $k = 1$ контексты совпадают, поэтому значения объединяются и теряется точность. При $k = 2$ контексты различаются, и анализ может хранить разные значения для разных путей выполнения программы.
