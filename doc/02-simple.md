## Задание 2. Добавить тип `Bool`

### Старые правила
| Конструкция       | Правило |
|---                |---      |
| $ I $             | $ [[I]] = int $ |
| $E_1 == E_2$      | $ [[E_1]] = [[E_2]] \land [[E_1 == E_2]] = int $ |
| $E_1 op E_2$      | $ [[E_1]] = [[E_2]] = [[E_1 op E_2]] = int $  |
| $ input $         | $ [[input]] = int $  |
| $ X = E $         | $ [[X]] = [[E]] $ |
| $ output E $      | $ [[E]] = int $ |
| $ if (E) {S}  $   | $ [[E]] = int $ |
| $ if (E) {S_1} else {S_2} $ | $ [[E]] = int $ |
| $ while (E) {S} $ | $ [[E]] = int $|
| $ f(X_1,...,X_n) {... return E;} $ | $ [[f]] = ([[X_1]],...,[[X_n]]) \rightarrow [[E]] $ |
| $ E(E_1,...,E_n) $ | $ [[E]] = ([[E_1]],...,[[E_n]]) \rightarrow [[(E)(E_1,...,E_n)]] $ |
| $ \&E $           | $ [[\&E]] = \&[[E]] $ |
| $ alloc $         | $ [[alloc]] = \&\alpha $ |
| $ null $          | $ [[null]] = \&\alpha $ |
| $ *E $            | $ [[E]] = \&[[*E]] $ |
| $ *X = E $        | $ [[X]] = \&[[E]] $ |


### Изменённые правила

- Разделим операторы на две группы
$ lop = \{==, >, <, \leq, \geq\} - логические\ операции$
$ aop = \{+, -, *, \div\} - арифметические\ операции$

| Конструкция       | Правило |
|---                |---      |
| $ I $             | $ [[I]] = int $ |
| $ true \lor false $ | $ [[true]] = bool \lor [[false]] = bool $ |
| $E_1\ lop\ E_2$   | $ [[E_1]] = [[E_2]] = int \land [[E_1 op E_2]] = bool $ |
| $E_1\ aop\ E_2$   | $ [[E_1]] = [[E_2]] = [[E_1 op E_2]] = int $  |
| $ input $         | $ [[input]] = int $  |
| $ X = E $         | $ [[X]] = [[E]] $ |
| $ output E $      | $ [[E]] = int $ |
| $ if (E) {S}  $   | $ [[E]] = bool $ |
| $ if (E) {S_1} else {S_2} $ | $ [[E]] = bool $ |
| $ while (E) {S} $ | $ [[E]] = bool $|
| $ f(X_1,...,X_n) {... return E;} $ | $ [[f]] = ([[X_1]],...,[[X_n]]) \rightarrow [[E]] $ |
| $ E(E_1,...,E_n) $ | $ [[E]] = ([[E_1]],...,[[E_n]]) \rightarrow [[(E)(E_1,...,E_n)]] $ |
| $ \&E $           | $ [[\&E]] = \&[[E]] $ |
| $ alloc $         | $ [[alloc]] = \&\alpha $ |
| $ null $          | $ [[null]] = \&\alpha $ |
| $ *E $            | $ [[E]] = \&[[*E]] $ |
| $ *X = E $        | $ [[X]] = \&[[E]] $ |


### Влияние на полноту

Полнота для анализа типов определяется насколько большая область slack. Для того, чтобы показать влияние на полноту надо показать пример программы, который будет либо отвергаться, когда в старой системе он принимался; либо приниматься, если старый анализ его отвергал.
Покажем что полнота уменьшается. Для этого надо привести пример программы которые пропускал старый анализ и не пропускает новый и доказать, что мы не можем привести пример, когда новый анализ принимает программу, которую отвергает старый анализ
Пример:
```tip
if (42) {
    output 42;
}
```
Новые правила для анализа тивоп потребует чтобы в условии if был тип bool, но литерал '42' является литералом с типом 'int' => новый анализ отвергнет эту программу, что следовательно увеличивает область slack. 

Обратный пример с тем, что новый анализ принимает тем программы которые раньше не принимал у нас нет.

Тем самым новый анализ уменьшает полноту анализа.

### Влияние на точность

```tip
main() {
    var x;
    x = 42 > input;
    x = x + 42;
    return x;
}
```

Новый тип не пропустит эту программу, потому что новый анализ выведет тип $[[x]] = bool$ т.к. тип выражения $42 > input$ - 'bool, а в следующей строчки мы делаем 'x + 42' что покажет несовпадения типов т.к. для '+' нужно чтобы оба операнда имели тип 'int'. Старый анали пропустил бы такую программу, т.к. результат логических и арифметических операций имели тип 'int'.

### Вывод

Добавление типа 'bool' уменьшает полноту, но повышает точность анализа.

## Задание 3. Добавить тип `Array`

### Новые правила

| Конструкция | Правило |
|---|---|
| $\{\}$ | $[[\{\}]]=\alpha[]$, где $\alpha$ — свежая типовая переменная |
| $\{E_1,\ldots,E_n\}$ | $[[E_1]]=\ldots=[[E_n]]=\alpha \land [[\{E_1,\ldots,E_n\}]]=\alpha[]$ |
| $X[I]$ | $[[X]]=\alpha[] \land [[I]]=int \land [[X[I]]]=\alpha$ |
| $X[I]=E$ | $[[X]]=\alpha[] \land [[I]]=int \land [[E]]=\alpha$ |

### Вывод

Добавление типа массива является консервативным расширением системы типов. Старые правила не изменяются, поэтому для любой программы $P$ старого языка, не содержащей массивов:

$$
P\in A_{old}
\Longleftrightarrow
P\in A_{arr}
$$

Следовательно, на старом фрагменте языка полнота и точность анализа не изменяются.

Для нового фрагмента языка предполагается, что массивы однородны, индекс имеет тип `int`, а записываемое значение имеет тип элемента массива. При такой runtime-семантике правила соответствуют языку. Если runtime допускает неоднородные массивы, требование единого типа элементов будет отвергать часть выполняемых программ и увеличивать область `slack`.

При этом простая система типов не проверяет выход индекса за границы массива.

## Задание 4. Протипизировать программу с массивами

```tip
main() {
    var x,y,z,t;
    x = {2, 4, 8, 16, 32, 64};
    y = x[x[3]];
    z = {{}, x};
    t = z[1];
    t[2] = y;
}
```

### Система ограничений

$$
\begin{cases}
[[2]]=\ldots=[[64]]=int
\\
[[\{2,4,8,16,32,64\}]]=int[]
\\
[[x]]=[[\{2,4,8,16,32,64\}]]
\\[4pt]

[[x]]=\alpha_1[]
\\
[[3]]=int
\\
[[x[3]]]=\alpha_1
\\

[[x]]=\alpha_2[]
\\
[[x[3]]]=int
\\
[[x[x[3]]]]=\alpha_2
\\
[[y]]=[[x[x[3]]]]
\\[4pt]

[[\{\}]]=\alpha_3[]
\\
[[\{\}]]=\alpha_4
\\
[[x]]=\alpha_4
\\
[[\{\{\},x\}]]=\alpha_4[]
\\
[[z]]=[[\{\{\},x\}]]
\\[4pt]

[[z]]=\alpha_5[]
\\
[[1]]=int
\\
[[z[1]]]=\alpha_5
\\
[[t]]=[[z[1]]]
\\[4pt]

[[t]]=\alpha_6[]
\\
[[2]]=int
\\
[[y]]=\alpha_6
\end{cases}
$$
### Итоговые типы

$$
[[x]]=int[] \\
[[y]]=int \\
[[z]]=int[][] \\
[[t]]=int[] \\
$$

### Вывод

Система ограничений успешно унифицируется. Выражение `x[3]` имеет тип `int`, поэтому оно может использоваться как индекс в `x[x[3]]`; результат внешнего индексирования также имеет тип `int`.

Тип пустого массива определяется из контекста. Поскольку `{}` находится в одном внешнем массиве с `x:int[]`, он получает тип `int[]`. Поэтому `z` имеет тип `int[][]`, а после `t=z[1]` переменная `t` имеет тип `int[]`.

Запись `t[2]=y` корректна, поскольку элементы `t` имеют тип `int`, а `y` также имеет тип `int`.
