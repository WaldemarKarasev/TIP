# Лекция 6. Path

## Пример, где анализ open/close всё равно ошибается (слайд 30)

Рассмотрим программу:

```tip
var x, flag, copy;

if (x > 15) {
    open();
    flag = 1;
} else {
    flag = 0;
}

copy = flag;

if (copy) {
    close();
}
```

Эта программа корректна: если файл открывается, то переменная `flag` становится равной `1`, затем значение `flag` копируется в `copy`, и условие `if (copy)` выполняется. Поэтому после `open()` обязательно будет вызван `close()`.

Если же ветка с `open()` не выполняется, то `flag = 0`, затем `copy = 0`, и второй `if` не выполнится. Значит, программа не пытается закрыть уже закрытый файл.

Сначала покажем, что программа корректна.

##### Случай 1. `x > 15`


Тогда выполняется ветка:

```tip
open();
flag = 1;
```

После этого:

```tip
copy = flag;
```

значит `copy = 1`, и условие `if (copy)` выполняется. В результате вызывается `close()`. То есть файл сначала открывается, а затем корректно закрывается.

##### Случай 2. `x <= 15`

Тогда выполняется ветка:

```tip
flag = 0;
```

После этого:

```tip
copy = flag;
```

значит `copy = 0`, и условие `if (copy)` не выполняется. В этом случае `close()` не вызывается, и попытки закрыть неоткрытый файл нет.

Таким образом, программа корректна: `close()` вызывается только тогда, когда ранее был вызван `open()`.

### Как анализ может ошибиться  

Анализ учитывает условия переходов, например:

```text
x > 15
copy == 1 
```

Но он может не учитывать зависимость между переменными, возникающую из присваивания:

```tip
copy = flag;
```

Если анализ не хранит факт `copy == flag`, он может рассмотреть невозможный путь:

```text
x <= 15, но copy == 1 
```

На таком пути файл не был открыт (так как `x <= 15`), но условие `if (copy)` считается истинным, и анализ делает вывод, что может выполниться `close()`. В результате появляется ложная ошибка: анализ считает, что возможно закрытие уже закрытого файла.


### Решение

Чтобы избежать такой ошибки, нужно расширить анализ так, чтобы он учитывал не только условия переходов, но и зависимости между переменными. Например, можно добавить в контекст пути предикаты:

```text
flag == 0
flag == 1
copy == 0
copy == 1
copy == flag
```

Тогда после присваивания `copy = flag` анализ сможет сохранить связь между переменными и отбросить невозможный путь, где `flag == 0`, но `copy == 1`.
